# RedSkill 上传检查单

## 上传文件

- 文件：`rousepet-redskill-v1.3.0.zip`
- 校验：同时保留 `.zip.sha256`
- 类型：原创 Skill
- 一篇笔记只挂载这一份 RedSkill
- 上传解析结果必须为 `42` 个文件，且不应提示过滤不支持的格式。若文件数不是 42，停止提交并重新检查压缩包。
- 包内顶层只能有 `rousepet/`，其中 `SKILL.md` 的 frontmatter 必须是 `name: rousepet`。

## 发布设置

- 公开发布，不使用私密或定时发布。
- 精确添加话题 `#REDSkill`，注意大小写。
- 若封面或正文图使用 AI 生成，手动勾选“含 AI 合成内容”。
- 不勾选商品或商业笔记，不在正文放 GitHub URL、二维码或其他站外引导。
- RedSkill v1.3.0 只标注 Windows 10/11 正式支持；不要在这次 Skill 提交中宣称支持 macOS/Linux。

## 上传后验证

1. 用另一个 Codex 环境获取 RedSkill。
2. 发送：“安装并立即显示 RousePet，保持我的官方 Codex 宠物完全不变。”
3. 确认 Codex 先完整披露六项本地权限，并等待用户明确同意；未同意时安装器必须拒绝执行。
4. 确认安装过程没有访问 GitHub、没有下载 EXE，也没有覆盖 PowerShell 执行策略。
5. 确认已有官方宠物 SHA-256 安装前后相同。
6. 确认桌宠立即出现；无宠物时出现创建入口。
7. 确认首次 Hooks 审核说明清楚，没有绕过安全确认。
8. 确认右键设置、缩放、久坐阶段和退出可用。
9. 对没有明确 `tired/sick/rest` 的宠物，确认桌宠先出现，再生成私有六帧动作；联系表通过后 `healthExtension.status=complete`，官方图集哈希保持不变。
