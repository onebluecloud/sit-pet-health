# RousePet v1.2.0

RousePet 现在会为缺少专用健康动作的宠物补齐私有动画，而不是长期重复使用同一段回退帧。

桌宠仍会立即出现：RousePet 先从原图集建立安全回退动作；随后只为缺少的 `tired`、`sick`、`rest` 生成六帧动画。Codex 会使用当前宠物作为身份参考，完成透明背景处理、帧完整性检查和联系表视觉审核，全部通过后才原子切换到私有扩展图集。

## 本版变化

- 区分“明确声明的健康动作”与标准图集中的推断回退帧。
- 只生成缺失动作，不重画已有动作，也不重新生成整只宠物。
- 新动作只写入 RousePet Plugin 私有 clone 的 `extensions` 目录。
- 加入色键自动选择、六帧布局约束、确定性质检、联系表审核和原子启用。
- 图片生成不可用或质检失败时继续使用回退动作，宠物不会消失。
- 保留 v1.1.1 的 RedSkill 上传格式兼容修复。

## 安全边界

- `CODEX_HOME/pets` 与 `~/.codex/pets` 始终只读。
- 生成任务使用私有 clone 的身份参考，不写回官方图集。
- 不读取或保存 Codex 任务正文，不向 RousePet 服务器上传数据。
- 不安装 EXE；最差状态是可逆休息，不是死亡或医疗指标。
